The application architecture will be designed around several different constraints.

The project requirements specify to use JAVA Servlets and JSP pages along with AJAX and Google Maps.
There are many different patterns that can be followed to implement the application using these technologies.
It is important to consider the different parts of the application.

  • The application will be designed in the Model View Controller pattern.
  • The data model will consist of the business logic objects that will beused in the application.
  • The View will be the part of the application that the users will see and interact with and will be implement with the JSP pages.
  • The controller will be responsible for handling incoming requests and sending back the appropriate view for the users. 

Based on this type of architecture, an application framework can possibly be utilized to help reduce the amount of software code that would be written and improve efficiency.

There are two candidate application frameworks that can help better integrate the MVC pattern.
Currently, Spring and Struts are being explored. 

Another part of the application architecture is addressing the data persistence. 
The application will be drawing data from a relational database. 
Extracting data from the database, and performing data persistence will be done by utilizing Hibernate, a thrid party object relational mapping tool.
