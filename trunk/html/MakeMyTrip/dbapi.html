SessionFactory

               In order to be able to load and store objects to and from the database, we created a HibernateUtil.java
               file that builds and stores the global SessionFactory object for easy access in our application code.
               A SessionFactory can open up new Sessions which represent a single-threaded unit of work.
               Before accessing the database, a function needs to obtain a new Session, start a transaction,
               access the database then commit any changes that may have been made. Below is an example
               of using the HibernateUtil SessionFactory:

               Session session = HibernateUtil.getSessionFactory().getCurrentSession();
               session.beginTransaction();
                       //database access
               session.getTransaction().commit();


       Hibernate Manager Classes

               Each POJO has a corresponding hibernate manager class that contains functions specific to the
               class that it manages.
               
               Below is a list of the manager classes and some of the functions they implement

               AirlineManager
               -        getAirlineCodeAndName()
                               // returns a list of all airline codes and names

               AirportManager
               -        getAirportCodeAndName()        
                               // returns a list of all airport codes and names
               -        getAirportLatLong(String code)        
                               // returns the lat and long values for all airports

               CustomerManager
               -        register(Customer c)                
                               // registers a customer by adding them to the database
               -        login(String username, String password)        
                               // customer login

               FlightManager        
               -        addFlight(Flight f)                
                               // adds a flight to the database
               -        removeFlight(Flight f)        
                               // removes a flight from the database
               -        getFlight(int flightNo)                
                               // returns a flight with the given flight number

               ItineraryManager
               -        getItinerary(String username, String status)        
                               // returns the itinerary for a given user with the given status
               -        reserve (String username, int flightNo)        
                               // adds a flight to the user's itinerary with a reserved status
               -        getReserved(String username)        
                               // returns the reserved flights for a given user
               -        getBooked(String username)                
                               // returns the booked flights for a given user
               -        getCanceled(String username)        
                               // returns the canceled flights for a given user

       Plain Old Java Object (POJO)

               Each table in the database is represented by a Plain Old Java Object
               (POJO) which has several constructors as well as getter and setter
               methods for every field in the table. These POJO files are all located
               in the 'hibernate' package of the project.
