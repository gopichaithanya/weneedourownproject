  • Spring Framework
               
       Learned a new application framework, the Spring framework and how it is connected to JSPs and
       Hibernate objects. This framework illustrated how to use MVC Architecture to build web applications.
       Learnt how Spring is used to post forms and bind the post parameters to server-side classes.

       Examined the spring framework a little closer to try to learn how the various requests are handled
       by the core servlet and spring xml file. We have found Spring Framework provides very nice and
       convenient Unit/Integration testing environment without any web container. It helped us to enhance
       the quality of our project. Another interesting functionality of Spring Framework is validator for
       form submitting. The validator functionality keeps the data persistence safe and robustic.

  • Java Server Pages

       We used JSPs to develop information-rich, dynamic Web pages. It aided us in separating the user
       interface from content generation, thus enabling us to change the overall page layout without
       altering the underlying dynamic content. By separating the page logic from its design and display and
       supporting a reusable component-based design, JSP makes Web applications faster and easier
       to build.

  • MVC Architecture

       We used the MVC (Model-View-Controller) architecture for designing the components. In our web
       application, MVC architecture is mainly used for separating business layer functionality represented
       by Spring Framework (the model) from the presentation layer functionality represented by JSPs
       (the view) using an intermediate servlet based controller.

  • Apache Tomcat Web Server
       
       We used Apache Tomact 6.0 Web server for the project. It provided a "pure Java" HTTP web server
       environment for Java code to run. It was easy to configure and use.

  • Hibernate, HyperSonic SQL and Java Database Connectivity
       
       Hibernate, database persistence framework, gave us an easy way to use JDBC. Used Hibernate as
       an ORM tool to illustrate the reuse of Database Objects. Hibernate also contributed to MVC
       architecture, because the database objects served as our models. Hibernate allowed us to map the
       database tables to our reusable objects. It efficiently handled all the database transactions. It was
       easily wired with HSQL, HyperSonic Database. It freed us to convert different data types between
       Java and Database.

  • Apache ANT

       Before this project, none of us knew how to use and maintain ANT. ANT build system gets the
       packaging and deploying process done automatically.  It expedited our developing speed.
       It worked on different Operating System in a same way.

  • Google SVN

       Google SVN was the Configuration Management tool used for Source Version Control. It helped us to
       synchronize the source code and roll them back in case of errors. It was well supported by different
       Java IDEs such as Eclipse and NetBeans.


  • Web Services

       We used Web services, to publish two functions to the rest of the world. We learnt the two main
       advantages of Web Services which are reusing application components and connect existing software.
       Using Web services we can exchange data between different applications and different platforms.

  • JavaScript and AJAX
       
       Used Javascript for Client-side validation. Learnt how AJAX can be used to fetch information from the server
       without reloading the page. The AJAX technique makes Internet applications smaller, faster and more
       user-friendly.

  • GoogleMaps APIs
       
       Used Google Maps API along with JavaScript to create Online Mashup.
